# Διαχωρισμός Ημερομηνίας Λήξης Εργασίας στο Aspose.Tasks

## Εισαγωγή
Η αποτελεσματική **διαχείριση χρονοδιαγραμμάτων έργων** αποτελεί θεμέλιο λίθο της επιτυχούς υλοποίησης. Όταν η διάρκεια μιας εργασίας αλλάζει, η ημερομηνία λήξης της πρέπει να επαναϋπολογιστεί ώστε το πρόγραμμα να παραμένει ακριβές. Σε αυτό το tutorial θα σας δείξουμε **πώς να διαχωρίσετε τις ημερομηνίες λήξης εργασιών** με το Aspose.Tasks for Java, παρέχοντάς σας ακριβή έλεγχο του χρονοδιαγράμματος του έργου σας.

## Γρήγορες Απαντήσεις
- **Τι επιτυγχάνει ο διαχωρισμός μιας ημερομηνίας λήξης εργασίας;** Σας επιτρέπει να υπολογίσετε την ημερομηνία λήξης για οποιαδήποτε διάρκεια, βοηθώντας σας να προσαρμόζετε τα προγράμματα άμεσα.  
- **Ποια βιβλιοθήκη απαιτείται;** Aspose.Tasks for Java (διαθέσιμη για λήψη από την επίσημη ιστοσελίδα).  
- **Χρειάζομαι άδεια για ανάπτυξη;** Μια προσωρινή άδεια λειτουργεί για δοκιμές· απαιτείται πλήρης άδεια για παραγωγή.  
- **Μπορώ να το χρησιμοποιήσω με οποιονδήποτε ημερολογιακό πίνακα του έργου;** Ναι, το API χρησιμοποιεί το ημερολόγιο του έργου για να σεβαστεί τις εργάσιμες ημέρες και τις αργίες.  
- **Πόσες γραμμές κώδικα χρειάζονται;** Λιγότερες από δέκα γραμμές για να λάβετε την ημερομηνία λήξης για οποιαδήποτε διάρκεια.

## Τι σημαίνει “διαχείριση χρονοδιαγραμμάτων έργων” στο πλαίσιο του Aspose.Tasks;
Η διαχείριση χρονοδιαγραμμάτων έργων σημαίνει τη διατήρηση των ημερομηνιών έναρξης και λήξης κάθε εργασίας συγχρονισμένες με το συνολικό πρόγραμμα, λαμβάνοντας υπόψη τα ημερολόγια, τους πόρους και τις εξαρτήσεις των εργασιών. Οι ακριβείς υπολογισμοί ημερομηνίας λήξης είναι απαραίτητοι για ρεαλιστικές προβλέψεις και την εμπιστοσύνη των ενδιαφερομένων.

## Γιατί να διαχωρίσετε την ημερομηνία λήξης μιας εργασίας;
- **Ευελιξία:** Δείτε γρήγορα πώς διαφορετικές κατανομές ωρών εργασίας επηρεάζουν την παράδοση.  
- **Αξιολόγηση κινδύνου:** Αξιολογήστε σενάρια “τι θα συνέβαινε αν” χωρίς να τροποποιήσετε το αρχικό σχέδιο.  
- **Προγραμματισμός πόρων:** Ευθυγραμμίστε τις διάρκειες εργασιών με τη χωρητικότητα της ομάδας και τους περιορισμούς του ημερολογίου.

## Εισαγωγή Πακέτων
Ξεκινήστε συμπεριλαμβάνοντας τα απαραίτητα πακέτα στο έργο Java σας:
```java
import com.aspose.tasks.*;
```

## Βήμα 1: Εύρεση Εργασίας προς Διαχωρισμό
Εντοπίστε την εργασία που θέλετε να διαχωρίσετε μέσα στο έργο:
```java
// The path to the documents directory.
String dataDir = "Your Document Directory";
// Read project
String projectName = dataDir + "SplitTask.mpp";
Project project = new Project(projectName);
Task splitTask = project.getRootTask().getChildren().getByUid(1);
```

## Βήμα 2: Εύρεση Ημερολογίου Έργου
Ανακτήστε το ημερολόγιο του έργου για ακριβείς υπολογισμούς ημερομηνιών:
```java
Calendar calendar = project.get(Prj.CALENDAR);
```

## Βήμα 3: Υπολογισμός Ημερομηνίας Λήξης Εργασίας με Διαφορετικές Διάρκειες
Τώρα, υπολογίστε την ημερομηνία λήξης της εργασίας για διάφορες διάρκειες.

### Βήμα 3.1: Διάρκεια 8 Ωρών
```java
System.out.println("Start Date: " + splitTask.get(Tsk.START) + "\n+ Duration 8 hours\nFinish Date: " + calendar.getTaskFinishDateFromDuration(splitTask, 8d));
```

*Επαναλάβετε τον παραπάνω κώδικα για διαφορετικές διάρκειες, προσαρμόζοντας τις ώρες ανάλογα, ώστε να δείτε πώς κάθε αλλαγή επηρεάζει την ημερομηνία λήξης.*

## Συχνά Προβλήματα και Λύσεις
- **Λανθασμένη αναφορά ημερολογίου:** Βεβαιωθείτε ότι ανακτάτε το ημερολόγιο του έργου (`project.get(Prj.CALENDAR)`) αντί να δημιουργείτε νέο.  
- **Λανθασμένο UID εργασίας:** Το UID πρέπει να αντιστοιχεί σε μια εργασία που υπάρχει πραγματικά· διαφορετικά το `getByUid` επιστρέφει `null` και προκαλεί `NullPointerException`.  
- **Ασυμφωνίες ζώνης ώρας:** Το API λειτουργεί με τη ζώνη ώρας του ημερολογίου· ελέγξτε τη ζώνη ώρας του συστήματός σας αν τα αποτελέσματα φαίνονται λανθασμένα.
