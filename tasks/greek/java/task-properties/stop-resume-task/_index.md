# Πώς να Επαναλάβετε Έργο και να Σταματήσετε Έργο στο Aspose.Tasks

## Εισαγωγή
Αν δημιουργείτε μια λύση διαχείρισης έργων βασισμένη σε Java, συχνά θα χρειαστεί να **παύσετε** ένα έργο προσωρινά και στη συνέχεια να το **συνεχίσετε** αργότερα. Το Aspose.Tasks for Java το κάνει εύκολο με ενσωματωμένες ιδιότητες για τη διακοπή και την επανέναρξη των έργων. Σε αυτό το σεμινάριο θα ανακαλύψετε **πώς να επαναλάβετε ένα έργο** και **πώς να σταματήσετε ένα έργο** προγραμματιστικά, και θα δείτε επίσης πώς να **φιλτράρετε τα έργα κατά ημερομηνία** ώστε να εργάζεστε μόνο με τα σχετικά στοιχεία στο πρόγραμμα σας.

## Γρήγορες Απαντήσεις
- **Τι σημαίνει το “stop” για ένα έργο;** Ορίζει μια ημερομηνία διακοπής, παύοντας την εργασία μετά από αυτό το σημείο.  
- **Πώς μπορώ να επαναλάβω ένα σταματημένο έργο;** Αναθέτοντας μια ημερομηνία επανέναρξης που είναι μεταγενέστερη από την ημερομηνία διακοπής.  
- **Μπορώ να περιορίσω τη λειτουργία σε ένα εύρος ημερομηνιών;** Ναι – χρησιμοποιήστε μια ελάχιστη ημερομηνία για να φιλτράρετε τα έργα.  
- **Ποια έκδοση της βιβλιοθήκης απαιτείται;** Οποιαδήποτε πρόσφατη έκδοση του Aspose.Tasks for Java (το API παραμένει σταθερό).  
- **Χρειάζομαι άδεια για ανάπτυξη;** Μια δωρεάν δοκιμή λειτουργεί για δοκιμές· απαιτείται άδεια για παραγωγή.

## Τι είναι το “how to resume task” στο Aspose.Tasks;
Η επανέναρξη ενός έργου σημαίνει απλώς την παροχή μιας ημερομηνίας **RESUME** σε ένα αντικείμενο `Task`. Όταν η μηχανή του έργου επεξεργάζεται το χρονοδιάγραμμα, η εργασία θα συνεχιστεί από εκείνη την ημερομηνία και μετά. Αυτό είναι ιδιαίτερα χρήσιμο για τη διαχείριση διακοπών όπως η μη διαθεσιμότητα πόρων ή εξωτερικές εξαρτήσεις.

## Γιατί να χρησιμοποιήσετε τη λειτουργία διακοπής/επανέναρξης;
- **Έλεγχος των χρονοδιαγραμμάτων:** Παύστε την εργασία χωρίς να διαγράψετε το έργο.  
- **Ακριβής αναφορά:** Εμφανίστε ρεαλιστικές ημερομηνίες έναρξης/λήξης σε διαγράμματα Gantt.  
- **Εύκολη αυτοματοποίηση:** Συνδυάστε με φίλτρα ημερομηνίας για να ενημερώσετε πολλά έργα ταυτόχρονα.

## Προαπαιτούμενα
- Καλή κατανόηση των βασικών της Java.  
- Εγκατεστημένο JDK στο σύστημά σας.  
- Η βιβλιοθήκη Aspose.Tasks for Java προστιθέμενη στο έργο σας (κατεβάστε την από [εδώ](https://releases.aspose.com/tasks/java/)).  

## Εισαγωγή Πακέτων
Πρώτα, φέρτε τις απαιτούμενες κλάσεις στο πεδίο ορατότητας ώστε να μπορείτε να εργάζεστε με έργα και εργασίες.

```java
import com.aspose.tasks.ChildTasksCollector;
import com.aspose.tasks.Project;
import com.aspose.tasks.Task;
import com.aspose.tasks.TaskUtils;
import com.aspose.tasks.Tsk;
import java.util.Calendar;
import java.util.GregorianCalendar;
```

## Βήμα 1: Αρχικοποίηση του Project και του Collector
Δημιουργήστε ένα αντικείμενο `Project` που δείχνει στο αρχείο MPP σας και ρυθμίστε έναν `ChildTasksCollector` για να συλλέξετε κάθε εργασία στην ιεραρχία.

```java
// The path to the documents directory.
String dataDir = "Your Document Directory";
Project project = new Project(dataDir + "Software Development.mpp");
ChildTasksCollector collector = new ChildTasksCollector();
TaskUtils.apply(project.getRootTask(), collector, 0);
```

## Βήμα 2: Ορισμός Ελάχιστης Ημερομηνίας για Φιλτράρισμα
Αν θέλετε να εργάζεστε μόνο με εργασίες που έχουν ουσιαστικές ημερομηνίες διακοπής ή επανέναρξης, ορίστε μια **ελάχιστη ημερομηνία**. Αυτό αποτελεί τον πυρήνα της τεχνικής *φιλτράρισμα εργασιών κατά ημερομηνία*.

```java
java.util.Date minDate = new GregorianCalendar(2000, Calendar.JANUARY, 1).getTime();
```

## Βήμα 3: Επανάληψη, Έλεγχος και Εμφάνιση Τιμών Διακοπής/Επανέναρξης
Τώρα επαναλάβετε μέσω των συλλεγμένων εργασιών, εφαρμόστε τη λογική **πώς να σταματήσετε ένα έργο** και εκτυπώστε τις ημερομηνίες. Ο κώδικας επίσης δείχνει **πώς να επαναλάβετε ένα έργο** διαβάζοντας την ιδιότητα `RESUME`.

```java
for (Task tsk : collector.getTasks()) {
    if (tsk.get(Tsk.STOP).before(minDate)) {
        System.out.println("NA");
    } else {
        System.out.println(tsk.get(Tsk.STOP).toString());
    }
    if (tsk.get(Tsk.RESUME).before(minDate)) {
        System.out.println("NA");
    } else {
        System.out.println(tsk.get(Tsk.RESUME).toString());
    }
}
```

> **Συμβουλή:** Αντικαταστήστε τις κλήσεις `System.out.println` με τη δική σας λογική (π.χ., ενημέρωση των ημερομηνιών, καταγραφή σε αρχείο ή τροποποίηση των αντικειμένων εργασίας) για να *σταματήσετε* ή *επανεκκινήσετε* τις εργασίες όπως απαιτείται.

## Συχνά Προβλήματα και Λύσεις
| Πρόβλημα | Αιτία | Διόρθωση |
|----------|-------|----------|
| `NullPointerException` στο `tsk.get(Tsk.STOP)` | Η εργασία δεν έχει ορισμένη ημερομηνία διακοπής. | Ελέγξτε για `null` πριν καλέσετε `.before(minDate)`. |
| Οι ημερομηνίες εμφανίζονται ως `NA` ακόμη και μετά τον ορισμό | Η ελάχιστη ημερομηνία είναι πολύ πρόσφατη. | Χρησιμοποιήστε μια πιο παλιά `minDate` ή αφαιρέστε το φίλτρο. |
| Οι αλλαγές δεν αντικατοπτρίζονται στο αποθηκευμένο MPP | Το έργο δεν αποθηκεύτηκε μετά τις τροποποιήσεις. | Καλέστε `project.save("output.mpp");` μετά την ενημέρωση των εργασιών. |
