# Πώς να αποθηκεύσετε εικόνες με το Aspose.Tasks για .NET

## Εισαγωγή

Σε αυτό το tutorial θα ανακαλύψετε **πώς να αποθηκεύσετε εικόνες** από αρχεία Microsoft Project χρησιμοποιώντας το Aspose.Tasks API για .NET. Είτε χρειάζεστε να ενσωματώσετε γραφήματα σε αναφορές, να δημιουργήσετε HTML σελίδες που περιέχουν οπτικά στοιχεία του έργου, είτε απλώς να εξάγετε διαγραμματικούς πόρους, τα παρακάτω βήματα θα σας καθοδηγήσουν σε όλη τη διαδικασία — από τη ρύθμιση του αντικειμένου Project μέχρι την προσαρμογή των callbacks εξαγωγής εικόνας.

## Γρήγορες Απαντήσεις
- **Τι σημαίνει “πώς να αποθηκεύσετε εικόνες” στο Aspose.Tasks;**  
  Αναφέρεται στη χρήση της διεπαφής `IImageSavingCallback` για τον έλεγχο του πού και πώς οι οπτικοί πόροι γράφονται στο δίσκο.
- **Μπορώ να αποθηκεύσω ένα έργο ως HTML με ενσωματωμένες εικόνες;**  
  Ναι, χρησιμοποιώντας το `HtmlSaveOptions` μαζί με callbacks αποθήκευσης εικόνας μπορείτε **να αποθηκεύσετε το έργο ως HTML** που περιλαμβάνει όλες τις παραγόμενες εικόνες.
- **Χρειάζεται άδεια για εξαγωγή εικόνας;**  
  Μια προσωρινή αξιολογική άδεια λειτουργεί για δοκιμές· απαιτείται πλήρης άδεια για παραγωγική χρήση.
- **Ποιες εκδόσεις .NET υποστηρίζονται;**  
  Το Aspose.Tasks για .NET υποστηρίζει .NET Framework 4.5+, .NET Core 3.1+, και .NET 5/6+.
- **Μπορεί να προσαρμοστεί η εξαγωγή εικόνας (μορφή, φάκελος, ονομασία);**  
  Απόλυτα – το callback σας δίνει πλήρη έλεγχο πάνω στο όνομα αρχείου, τη μορφή και τον προορισμό.

## Τι σημαίνει “πώς να αποθηκεύσετε εικόνες” στο πλαίσιο του Aspose.Tasks;
Η αποθήκευση εικόνων σημαίνει εξαγωγή οπτικών στοιχείων (γράφημα, ράβδοι Gantt, γραφικά πόρων) από ένα αρχείο Project και εγγραφή τους σε αρχεία εικόνας (PNG, JPEG κ.λπ.). Το Aspose.Tasks παρέχει έναν ευέλικτο μηχανισμό callback που σας επιτρέπει να αποφασίσετε την ακριβή τοποθεσία, τη σύμβαση ονομασίας και ακόμη και τη μορφή της εικόνας.

## Γιατί να χρησιμοποιήσετε το Aspose.Tasks για αποθήκευση εικόνων;
- **Πλήρης προγραμματιστικός έλεγχος** – δεν απαιτείται χειροκίνητη αλληλεπίδραση UI.  
- **Δια-πλατφορμική** – λειτουργεί σε Windows, Linux και macOS μέσω .NET Core.  
- **Υψηλής πιστότητας απόδοση** – οι εικόνες διατηρούν την ίδια ποιότητα με την αρχική προβολή του Project.  
- **Εύκολη δημιουργία HTML** – συνδυάστε το `HtmlSaveOptions` με callbacks εικόνας για **αυτόματη δημιουργία HTML με εικόνες**.

3. Βασική εξοικείωση με τη γλώσσα C# και τη δομή έργου .NET.

## Εισαγωγή Namespaces

Πρώτα, φέρτε τα απαιτούμενα namespaces στο αρχείο πηγαίου κώδικα:

```csharp
using Aspose.Tasks;
using System.Collections.Generic;
using System.IO;

using Aspose.Tasks.Saving;
using Aspose.Tasks.Visualization;
```

## Βήμα 1: Δημιουργία Αντικειμένου Project

Φορτώστε το αρχείο Microsoft Project με το οποίο θέλετε να εργαστείτε:

```csharp
var project = new Project("Project1.mpp");
```

## Βήμα 2: Ορισμός Επιλογών Αποθήκευσης

Δημιουργήστε τις επιλογές αποθήκευσης HTML που θα περιέχουν επίσης τα callbacks αποθήκευσης εικόνας:

```csharp
var options = GetSaveOptions(1);
```

## Βήμα 3: Αποθήκευση του Project ως HTML (save project as html)

Τώρα εξάγετε το έργο σε αρχείο HTML. Οι εικόνες που αναφέρονται στο HTML θα παραχθούν από το callback που θα ορίσουμε στη συνέχεια:

```csharp
project.Save("document_out.html", options);
```

## Βήμα 4: Υλοποίηση Callback Αποθήκευσης Εικόνας (customize image export)

Υλοποιήστε τη διεπαφή `IImageSavingCallback`. Εδώ είναι που **προσαρμόζετε τη συμπεριφορά εξαγωγής εικόνας**:

```csharp
private class ResourcePrefixForNestedResources : IImageSavingCallback
{
    public void ImageSaving(ImageSavingArgs args)
    {
        // Image saving logic goes here
    }
}
```

## Βήμα 5: Αποθήκευση Εικόνων σε Καθορισμένο Κατάλογο

Μέσα στη μέθοδο `ImageSaving`, αποφασίστε πού θα αποθηκευτεί κάθε εικόνα. Το παρακάτω παράδειγμα διακρίνει πόρους PNG από άλλες μορφές:

```csharp
if (args.FileName.EndsWith("png"))
{
    // Save nested resources
}
else
{
    // Save regular resources
}
```

## Βήμα 6: Καθορισμός Επιλογών Αποθήκευσης (συμπεριλαμβανομένων των callbacks)

Συνδέστε τα callbacks για γραμματοσειρές, CSS και εικόνες. Αυτό εξασφαλίζει ότι κάθε οπτικό στοιχείο διαχειρίζεται συνεπώς:

```csharp
public static HtmlSaveOptions GetSaveOptions(int pageNumber)
{
    var options = new HtmlSaveOptions
    {
        // Specify save options here
    };

    var program = new ResourcePrefixForNestedResources();
    options.FontSavingCallback = program;
    options.CssSavingCallback = program;
    options.ImageSavingCallback = program;

    return options;
}
```

## Συνηθισμένα Προβλήματα και Λύσεις

| Πρόβλημα | Αιτία | Διόρθωση |
|----------|-------|----------|
| Οι εικόνες δεν εμφανίζονται στο παραγόμενο HTML | Το callback δεν αντιστοιχίζει έγκυρη διαδρομή αρχείου | Βεβαιωθείτε ότι το `args.Path` δείχνει σε φάκελο με δικαιώματα εγγραφής και ορίστε σωστά το `args.ImageStream`. |
| Τα αρχεία PNG αποθηκεύονται με λανθασμένη επέκταση | Η λογική ελέγχει μόνο το επίθημα “png” | Χρησιμοποιήστε `Path.GetExtension(args.FileName).Equals(".png", StringComparison.OrdinalIgnoreCase)` για πιο αξιόπιστο έλεγχο. |
| Οι αναφορές HTML σπάζουν μετά τη μετακίνηση αρχείων | Οι σχετικές διαδρομές άλλαξαν μετά τη μετακίνηση του φακέλου εξόδου | Κρατήστε τους φακέλους HTML και εικόνων μαζί ή ενημερώστε το `options.ImageFolder` αναλόγως. |
