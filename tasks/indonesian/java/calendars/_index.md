## What is “defining weekdays” in MS Project?
Definisi hari kerja berarti menentukan hari mana dalam seminggu yang dianggap hari kerja dan menetapkan waktu mulai/selesai, aturan lembur, atau pengecualian untuk setiap hari. Informasi ini menggerakkan penjadwalan tugas, alokasi sumber daya, dan perhitungan jalur kritis sepanjang siklus hidup proyek.

## Why use Aspose.Tasks for Java to define weekdays?
- **Full API control** – Tanpa batasan UI; Anda dapat secara programatis membuat, memodifikasi, atau menghapus entri hari kerja.
- **Cross‑platform** – Berfungsi pada lingkungan apa pun yang kompatibel dengan JVM, mulai dari aplikasi desktop hingga layanan cloud.
- **Precision** – Atur waktu kerja yang berbeda untuk setiap hari kerja, tambahkan pengecualian untuk hari libur, dan sinkronkan kalender di beberapa proyek.
- **Performance** – Manipulasi file proyek besar tanpa memuat seluruh UI, ideal untuk pipeline otomatis.

## Frequently Asked Questions

**Q: Bisakah saya mendefinisikan jam kerja yang berbeda untuk setiap hari kerja?**  
A: Ya. Aspose.Tasks memungkinkan Anda mengatur waktu mulai dan selesai secara terpisah untuk Senin hingga Minggu.

**Q: Bagaimana cara menangani hari libur atau hari non‑kerja?**  
A: Setelah mendefinisikan hari kerja, Anda dapat menambahkan pengecualian (tanggal) untuk menandai hari libur atau periode non‑kerja khusus.

**Q: Apakah memungkinkan menyalin definisi hari kerja dari satu kalender ke kalender lain?**  
A: Tentu saja. Anda dapat mengambil objek `WeekDay` dari kalender yang ada dan menambahkannya ke instance kalender lain.

**Q: Apakah saya perlu memuat ulang proyek setelah memperbarui hari kerja?**  
A: Tidak. Perubahan diterapkan langsung ke objek `Project` dalam memori; cukup simpan proyek saat selesai.

**Q: Versi Aspose.Tasks mana yang diperlukan untuk manipulasi hari kerja?**  
A: Semua versi terbaru (20.10 dan setelahnya) mendukung API hari kerja lengkap. Kami menyarankan menggunakan rilis stabil terbaru untuk kinerja terbaik.
