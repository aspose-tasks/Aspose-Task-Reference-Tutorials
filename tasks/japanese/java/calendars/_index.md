# Aspose.Tasks でカレンダーを作成する方法

## はじめに

プロジェクトスケジュールを駆動する **カレンダーの作成方法** を探している Java 開発者の方へ、ようこそ。このハブでは、MS Project のカレンダー管理に関する一連の Aspose.Tasks チュートリアルをまとめています。基本的なカレンダーの作成から、稼働時間の取得、平日の定義、さらにはカレンダーを MPP 形式に更新する方法まで、ステップバイステップで解説します。新しいスケジューリングエンジンを構築する場合でも、既存ソリューションを拡張する場合でも、これらのガイドが迅速かつ確実に作業を進める手助けをします。

## クイック回答
- **カレンダー操作の主要クラスは何ですか？** `com.aspose.tasks.Calendar`
- **既存のカレンダーから稼働時間を取得できますか？** はい – `WorkingTime` コレクションを使用します。
- **カスタム平日を定義できますか？** もちろんです。各日の `WeekDay` オブジェクトを設定します。
- **カレンダーを MPP 形式に更新するには？** `project.save("file.mpp", SaveFileFormat.MPP)` を呼び出します。
- **本番環境でライセンスは必要ですか？** 評価版以外のデプロイには有料の Aspose.Tasks ライセンスが必要です。

## Aspose.Tasks の **カレンダー作成方法** とは？
Aspose.Tasks でカレンダーを作成するとは、`Calendar` オブジェクトをインスタンス化し、`Project` に添付して稼働日、祝日、例外を設定することを意味します。このオブジェクトは、MS Project ファイル内のすべてのタスクスケジューリング計算の基盤となります。

## Java のカレンダー管理に Aspose.Tasks を使用する理由
- **完全な MS Project 互換性** – カレンダーは忠実に保存・読み込みできます。
- **細かな制御** – 稼働時間のプログラム的調整、例外の追加、祝日の操作が可能です。
- **COM 相互運用不要** – 純粋な Java で、クロスプラットフォーム環境に最適です。
- **パフォーマンス最適化 API** – 大規模プロジェクトファイルでもメモリ使用量を最小限に抑えて処理できます。

## よくある質問

**Q: Aspose.Tasks で **カレンダーの作成方法** オブジェクトをプログラム的に作成するには？**  
A: `com.aspose.tasks.Calendar` をインスタンス化し、`project.getCalendars().add(calendar)` で `Project` に追加し、`WeekDays`、`Exceptions`、`WorkingTimes` を設定します。

**Q: 既存のカレンダーから **稼働時間を抽出する** ベストな方法は？**  
A: `Calendar.getWorkingTimes()` コレクションを使用します。各 `WorkingTime` エントリには、対象日の開始時刻と終了時刻が含まれます。

**Q: カスタム作業週の **平日を定義** できますか？**  
A: はい。カレンダーから `WeekDay` オブジェクトを取得し（`calendar.getWeekDays()`）、必要な各日について `DayWorking` と `WorkingTimes` を設定します。

**Q: カレンダーを変更した後、 **MPP ファイルを更新** する簡単な方法はありますか？**  
A: カレンダーを修正した後、`project.save("UpdatedProject.mpp", SaveFileFormat.MPP)` を呼び出して変更を MPP ファイルに書き込みます。

**Q: カレンダー操作ごとに別々のライセンスが必要ですか？**  
A: いいえ。単一の Aspose.Tasks ライセンスで、作成、抽出、定義、MPP 更新を含むすべてのカレンダー関連 API がカバーされます。
