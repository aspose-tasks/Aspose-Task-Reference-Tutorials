# Aspose.Tasks로 캘린더 만들기

## 소개

프로젝트 일정 관리를 주도하는 **how to create calendar** 객체를 만들고자 하는 Java 개발자라면, 바로 여기가 정답입니다. 이 허브에서는 기본 캘린더 생성부터 작업 시간 추출, 평일 정의, 그리고 캘린더를 MPP 형식으로 업데이트하는 등, MS Project 캘린더 관리의 모든 측면을 다루는 Aspose.Tasks 튜토리얼 시리즈를 모았습니다. 새로운 스케줄링 엔진을 구축하든 기존 솔루션을 확장하든, 단계별 가이드를 통해 빠르고 안정적으로 작업을 수행할 수 있습니다.

## 빠른 답변
- **캘린더 처리를 위한 기본 클래스는 무엇인가요?** `com.aspose.tasks.Calendar`
- **기존 캘린더에서 작업 시간을 추출할 수 있나요?** 예 – `WorkingTime` 컬렉션을 사용합니다.
- **사용자 정의 평일을 정의할 수 있나요?** 물론입니다, 각 요일에 대해 `WeekDay` 객체를 설정하면 됩니다.
- **캘린더를 MPP 형식으로 업데이트하려면 어떻게 하나요?** `project.save("file.mpp", SaveFileFormat.MPP)`를 호출합니다.
- **프로덕션 사용에 라이선스가 필요합니까?** 평가용이 아닌 배포에는 유료 Aspose.Tasks 라이선스가 필요합니다.

## Aspose.Tasks에서 **how to create calendar**란 무엇인가요?
Aspose.Tasks에서 캘린더를 만든다는 것은 `Calendar` 객체를 인스턴스화하고 이를 `Project`에 연결한 뒤 작업일, 휴일 및 예외를 설정하는 것을 의미합니다. 이 객체는 MS Project 파일 내 모든 작업 일정 계산의 핵심이 됩니다.

## Java 캘린더 관리를 위해 Aspose.Tasks를 사용하는 이유
- **전체 MS Project 호환성** – 캘린더가 손실 없이 저장 및 읽혀집니다.
- **세밀한 제어** – 작업 시간을 프로그래밍 방식으로 조정하고, 예외를 추가하며, 휴일을 관리할 수 있습니다.
- **COM 인터옵 없음** – 순수 Java이며, 크로스 플랫폼 환경에 이상적입니다.
- **성능 최적화 API** – 최소 메모리 사용량으로 대형 프로젝트 파일을 처리합니다.

## 자주 묻는 질문

**Q: Aspose.Tasks에서 **how to create calendar** 객체를 프로그래밍 방식으로 만들려면 어떻게 해야 하나요?**  
A: `com.aspose.tasks.Calendar`를 인스턴스화하고, `project.getCalendars().add(calendar)`를 통해 `Project`에 추가한 뒤, `WeekDays`, `Exceptions`, `WorkingTimes`를 구성합니다.

**Q: 기존 캘린더에서 **how to extract hours**를 추출하는 가장 좋은 방법은 무엇인가요?**  
A: `Calendar.getWorkingTimes()` 컬렉션을 사용합니다; 각 `WorkingTime` 항목에는 해당 날짜의 시작 및 종료 시간이 포함됩니다.

**Q: 사용자 정의 작업 주에 대해 **how to define weekdays**를 설정할 수 있나요?**  
A: 가능합니다. 캘린더에서 `WeekDay` 객체들을 (`calendar.getWeekDays()`) 가져와서, 필요에 따라 각 요일의 `DayWorking` 및 `WorkingTimes`를 설정합니다.

**Q: 캘린더를 변경한 후 **how to update mpp** 파일을 간단히 업데이트하는 방법이 있나요?**  
A: 캘린더를 수정한 뒤 `project.save("UpdatedProject.mpp", SaveFileFormat.MPP)`를 호출하여 변경 사항을 MPP 파일에 저장합니다.

**Q: 각 캘린더 작업마다 별도의 라이선스가 필요합니까?**  
A: 필요하지 않습니다. 하나의 Aspose.Tasks 라이선스로 생성, 추출, 정의, MPP 업데이트 등 모든 캘린더 관련 API를 사용할 수 있습니다.
