# Мониторинг стоимости проекта с Aspose.Tasks: Сверхурочная работа и труд

## Введение
В этом учебнике вы узнаете, как выполнять **мониторинг стоимости проекта** с помощью Aspose.Tasks for Java. Мы пройдем процесс отслеживания сверхурочных, оставшихся затрат и работы, чтобы вы могли держать свои проекты в графике и в рамках бюджета. Независимо от того, являетесь ли вы менеджером проекта или руководителем команды, эти шаги помогут вам поддерживать четкую видимость финансовых и ресурсных показателей.

## Быстрые ответы
- **Что я могу мониторить?** Стоимость сверхурочных, сверхурочная работа, оставшиеся затраты, оставшаяся работа и оставшиеся затраты на сверхурочные.  
- **Какая библиотека требуется?** Aspose.Tasks for Java.  
- **Нужна ли лицензия для разработки?** Для тестирования подходит бесплатная пробная версия; для продакшна требуется лицензия.  
- **Можно ли загрузить существующие файлы .mpp?** Да, просто укажите путь к файлу.  
- **Достаточно ли Java 6?** API поддерживает Java SE 6 и более новые версии.

## Что такое мониторинг стоимости проекта?
Мониторинг стоимости проекта — это практика непрерывного отслеживания всех финансовых аспектов проекта: запланированных затрат, фактических расходов и прогнозируемых оставшихся затрат. Интегрируя это с назначениями ресурсов, вы получаете информацию в реальном времени о расходах на сверхурочную работу и прогрессе выполнения задач.

## Почему стоит мониторить сверхурочную работу и оставшуюся работу?
- **Контроль бюджета:** Сверхурочная работа часто приводит к неожиданным перерасходам.  
- **Улучшение прогнозирования:** Знание оставшейся работы помогает проактивно корректировать графики.  
- **Повышение прозрачности:** Заинтересованные стороны могут точно видеть, где потребляются ресурсы.

## Предварительные требования
Прежде чем начать, убедитесь, что у вас есть следующее:
1. **Java Development Kit (JDK):** Aspose.Tasks for Java требует Java SE 6 или более новую версию.  
2. **Aspose.Tasks for Java Library:** Скачайте и установите библиотеку по ссылке [здесь](https://releases.aspose.com/tasks/java/).  
3. **Integrated Development Environment (IDE):** Любая Java IDE, например Eclipse, IntelliJ IDEA или NetBeans.

## Импорт пакетов
Начните с импорта необходимых пакетов в ваш Java‑файл:
```java
import com.aspose.tasks.Asn;
import com.aspose.tasks.Project;
import com.aspose.tasks.ResourceAssignment;
```

## Шаг 1: Настройка каталога данных
Определите каталог, в котором находится ваш файл проекта:
```java
String dataDir = "Your Data Directory";
```
Замените `"Your Data Directory"` на путь к вашему файлу проекта.

## Шаг 2: Загрузка проекта
Создайте объект `Project` и загрузите файл проекта:
```java
Project project = new Project(dataDir + "ResourceAssignmentOvertimes.mpp");
```
Замените `"ResourceAssignmentOvertimes.mpp"` на имя вашего файла MPP. Этот шаг демонстрирует использование **load mpp file**.

## Шаг 3: Итерация по назначениям ресурсов
Пройдите по каждому назначению ресурса в проекте:
```java
for (ResourceAssignment ra : project.getResourceAssignments()) {
```

## Шаг 4: Вывод стоимости сверхурочных и работы
Получите и выведите стоимость сверхурочных и работу для каждого назначения ресурса:
```java
    System.out.println(ra.get(Asn.OVERTIME_COST));
    System.out.println(ra.get(Asn.OVERTIME_WORK).toString());
```

## Шаг 5: Вывод оставшихся затрат и работы
Получите и выведите оставшиеся затраты и работу для каждого назначения ресурса:
```java
    System.out.println(ra.get(Asn.REMAINING_COST));
    System.out.println(ra.get(Asn.REMAINING_WORK).toString());
```

## Шаг 6: Вывод оставшихся затрат на сверхурочные и работы
Получите и выведите оставшиеся затраты на сверхурочные и работу для каждого назначения ресурса:
```java
    System.out.println(ra.get(Asn.REMAINING_OVERTIME_COST));
    System.out.println(ra.get(Asn.REMAINING_OVERTIME_WORK).toString());
}
```

## Распространённые проблемы и решения
- **Файл не найден:** Проверьте путь `dataDir` и убедитесь, что имя файла MPP указано правильно.  
- **Null values:** Некоторые назначения могут не содержать данных о сверхурочных; проверяйте `null` при выводе.  
- **Несоответствие версии:** Используйте версию библиотеки, соответствующую формату файла MPP (например, более новые версии MS Project).
