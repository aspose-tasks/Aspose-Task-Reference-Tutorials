# Запланированная vs Фактическая работа: Эффективное управление отклонениями проекта с Aspose.Tasks

## Introduction
В руководстве вы узнаете **как получать данные об отклонениях** и сравнивать *запланированную vs фактическую работу* с помощью Aspose.Tasks Java API. Отклонения — будь то работа, стоимость, даты начала или окончания — являются ключевыми индикаторами состояния графика. К концу этого руководства вы сможете рассчитывать отклонение стоимости, извлекать значения отклонений для каждой назначения ресурса и эффективно управлять отклонениями проекта, чтобы держать его в рамках плана.

## Quick Answers
- **Что такое “запланированная vs фактическая работа”?** Это разница между изначально запланированной работой и фактически выполненной работой.  
- **Какой класс API предоставляет данные об отклонениях?** Класс `Asn` (например, `Asn.WORK_VARIANCE`, `Asn.COST_VARIANCE`).  
- **Нужна ли лицензия для запуска примера?** Бесплатная пробная версия подходит для оценки; для производства требуется коммерческая лицензия.  
- **Можно ли получить все типы отклонений в одном цикле?** Да — перебирайте объекты `ResourceAssignment` и вызывайте `ra.get(Asn.*_VARIANCE)`.  
- **Какая версия Java требуется?** Рекомендуется Java 8 или выше.

## Prerequisites
Прежде чем продолжить, убедитесь, что у вас есть следующие предпосылки:
1. Установлен Java Development Kit (JDK) на вашей системе.  
2. Скачана библиотека Aspose.Tasks for Java и добавлена в ваш проект. Вы можете скачать её [здесь](https://releases.aspose.com/tasks/java/).  
3. Базовые знания языка программирования Java.

## Import Packages
Сначала импортируйте необходимые пакеты для работы с Aspose.Tasks:

```java
import com.aspose.tasks.Asn;
import com.aspose.tasks.Project;
import com.aspose.tasks.ResourceAssignment;
```

## Step 1: Iterate through Resource Assignments
Чтобы **управлять отклонениями проекта**, нам нужно пройтись по каждому назначению ресурса в загруженном файле проекта:

```java
// The path to the documents directory.
String dataDir = "Your Data Directory";
Project project = new Project(dataDir + "ResourceAssignmentVariance.mpp");
for (ResourceAssignment ra : project.getResourceAssignments()) {
    // Perform operations on each resource assignment
}
```

## Step 2: Retrieve Work Variance (Planned vs Actual Work)
Отклонение работы показывает разрыв между **запланированной vs фактической работой**. Получить его можно так:

```java
System.out.println(ra.get(Asn.WORK_VARIANCE));
```

## Step 3: Retrieve Cost Variance
Если вам нужно **рассчитать отклонение стоимости**, используйте следующий вызов:

```java
System.out.println(ra.get(Asn.COST_VARIANCE));
```

## Step 4: Retrieve Start Variance
Отклонение начала указывает разницу между запланированной датой начала и фактической датой начала:

```java
System.out.println(ra.get(Asn.START_VARIANCE));
```

## Step 5: Retrieve Finish Variance
Отклонение завершения отражает отклонение между запланированной датой завершения и фактической датой завершения:

```java
System.out.println(ra.get(Asn.FINISH_VARIANCE));
```

## Why Retrieve Variance Data?
Понимание отклонений помогает вам:
- Раннее обнаруживать отклонения графика.  
- Корректировать распределение ресурсов до роста расходов.  
- Формировать точные отчёты о статусе для заинтересованных сторон.  
- Проводить анализ причин задержек задач.

## Common Pitfalls & Tips
- **Pitfall:** Забыл загрузить правильный путь к файлу `.mpp`.  
  **Tip:** Убедитесь, что `dataDir` указывает на папку, содержащую `ResourceAssignmentVariance.mpp`.  
- **Pitfall:** Предположение, что значения отклонений всегда числовые.  
  **Tip:** Некоторые назначения могут возвращать `null`, если данные недоступны; добавьте проверки на `null`.  
- **Pro tip:** Используйте `ra.get(Asn.WORK)` вместе с `ra.get(Asn.WORK_VARIANCE)`, чтобы вычислить фактически выполненную работу.

Обработка **запланированной vs фактической работы** и других отклонений имеет решающее значение для эффективного контроля проекта. ## Frequently Asked Questions

**Q: Как рассчитать отклонение стоимости для конкретной задачи?**  
A: Используйте `ra.get(Asn.COST_VARIANCE)` после перебора назначений; комбинируйте с `ra.get(Asn.COST)` для полного анализа стоимости.

**Q: Что делать, если значение отклонения возвращает null?**  
A: `null` указывает, что данные не заданы в исходном файле. Защитите код проверкой на `null` перед выводом или использованием значения.

**Q: Можно ли экспортировать данные об отклонениях в Excel?**  
A: Да — соберите значения в коллекцию и используйте библиотеку, например Apache POI, чтобы записать их в лист Excel.

**Q: Поддерживает ли Aspose.Tasks метрики отклонений в Agile‑проектах?**  
A: Хотя API ориентирован на традиционные данные MS‑Project, вы можете сопоставить информацию о спринтах Agile с задачами и всё равно получать значения отклонений.

**Q: Есть ли способ пакетно обновлять значения отклонений?**  
A: Вы можете изменить базовые поля (например, `Asn.WORK`) и затем сохранить проект; поля отклонений пересчитаются при следующем чтении.
