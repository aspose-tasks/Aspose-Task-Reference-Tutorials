# Как добавить атрибут – Добавление расширенных атрибутов к задачам в Aspose.Tasks

## Введение
Повышение возможностей управления проектами имеет решающее значение для эффективного отслеживания задач и управления ресурсами. В этом руководстве **вы узнаете, как добавить атрибут** к задачам с помощью Aspose.Tasks for Java, получая гибкость настройки файлов Microsoft Project под конкретные потребности вашей организации. Мы пройдем весь процесс шаг за шагом, от настройки среды до сохранения обновленного файла проекта.

## Быстрые ответы
- **Какова основная цель?** Добавить пользовательские (расширенные) атрибуты к задачам в файле Microsoft Project.  
- **Какая библиотека требуется?** Aspose.Tasks for Java.  
- **Нужна ли лицензия?** Бесплатная пробная версия подходит для разработки; коммерческая лицензия требуется для продакшна.  
- **Можно ли добавить значения справочника?** Да — вы можете создавать текстовые или длительные атрибуты с предопределенными списками справочников.  
- **Какие форматы файлов поддерживаются для сохранения?** MPP, XML и другие форматы, поддерживаемые Aspose.Tasks.

## Предварительные требования
Прежде чем приступить к руководству, убедитесь, что у вас есть следующие предварительные требования:
- Базовые знания программирования на Java.  
- Установлена библиотека Aspose.Tasks for Java. Вы можете скачать её с [веб‑сайта](https://releases.aspose.com/tasks/java/).  
- На системе установлен интегрированный набор средств разработки Java (IDE).

## Импорт пакетов
В вашем Java‑проекте импортируйте необходимые пакеты для доступа к функционалу Aspose.Tasks:
```java
import java.io.IOException;
import com.aspose.tasks.*;
```

Теперь разберём каждый пример по нескольким шагам:

## Как добавить атрибут к задаче (пример с обычным текстом)
### 1. Установка каталога документов
Сначала определите папку, содержащую ваш исходный файл проекта:
```java
String dataDir = "Your Document Directory";
```

### 2. Загрузка проекта
Создайте экземпляр `Project`, указывающий на существующий файл `.mpp`:
```java
Project project = new Project(dataDir + "project.mpp");
```

### 3. Определение расширенного атрибута типа обычный текст
Создайте определение атрибута типа **Text1** — он будет хранить название города для каждой задачи:
```java
ExtendedAttributeDefinition taskExtendedAttributeText1Definition = ExtendedAttributeDefinition.createTaskDefinition(CustomFieldType.Text, ExtendedAttributeTask.Text1, "Task City Name");
```

### 4. Добавление определения в проект
Зарегистрируйте новое определение в коллекции расширенных атрибутов проекта:
```java
project.getExtendedAttributes().add(taskExtendedAttributeText1Definition);
```

### 5. Создание новой задачи
Добавьте дочернюю задачу к корневой задаче:
```java
Task task = project.getRootTask().getChildren().add("Task 1");
```

### 6. Создание экземпляра расширенного атрибута
Создайте реальный экземпляр атрибута из ранее созданного определения:
```java
ExtendedAttribute taskExtendedAttributeText1 = taskExtendedAttributeText1Definition.createExtendedAttribute();
```

### 7. Присвоение значения
Установите конкретное значение (например, город «London») для расширенного атрибута этой задачи:
```java
taskExtendedAttributeText1.setTextValue("London");
```

### 8. Привязка атрибута к задаче
Добавьте заполненный атрибут в коллекцию атрибутов задачи:
```java
task.getExtendedAttributes().add(taskExtendedAttributeText1);
```

### 9. Сохранение обновлённого проекта
Наконец, сохраните изменения в новый файл, чтобы вы могли проверить результат в Microsoft Project:
```java
project.save(dataDir + "PlainTextExtendedAttribute_out.mpp", SaveFileFormat.Mpp);
```

## Добавление текстового атрибута с опцией справочника
Чтобы создать текстовый атрибут, предлагающий предопределённый список значений (справочник), повторите вышеуказанные шаги, но замените `ExtendedAttributeTask.Text1` на `ExtendedAttributeTask.Text2` и заполните таблицу справочника с помощью `taskExtendedAttributeText2Definition.getLookupValues().add(...)`. Это позволяет пользователям выбирать из фиксированного набора вариантов при редактировании задачи.

## Добавление атрибута длительности с опцией справочника
Аналогично, для атрибута на основе длительности используйте `CustomFieldType.Duration` и `ExtendedAttributeTask.Duration2`. Заполните значения справочника конкретными временными интервалами (например, «1 day», «2 weeks»), чтобы стандартизировать ввод длительностей в вашем проекте.

## Почему использовать расширенные атрибуты?
- **Гибкость:** Хранить любые пользовательские данные, которые не покрыты стандартными полями Project.  
- **Стандартизация:** Таблицы справочников обеспечивают согласованный ввод данных.  
- **Отчётность:** Пользовательские поля могут быть включены в отчёты и фильтры, предоставляя более глубокие аналитические данные.

## Распространённые проблемы и устранение неполадок
- **Неверный путь:** Убедитесь, что `dataDir` заканчивается разделителем файлов (`/` или `\\`), соответствующим вашей ОС.  
- **Исключения лицензии:** Без действующей лицензии библиотека работает в режиме оценки и может добавлять водяные знаки.  
- **Справочник не отображается:** После определения значений справочника всегда добавляйте определение в проект перед созданием атрибутов задачи.
