# Как возобновить задачу и остановить задачу в Aspose.Tasks

## Введение
Если вы разрабатываете решение для управления проектами на Java, вам часто понадобится **приостановить** задачу временно, а затем **возобновить** её позже. Aspose.Tasks for Java упрощает это с помощью встроенных свойств для остановки и возобновления задач. В этом руководстве вы узнаете **как возобновить задачу** и **как остановить задачу** программно, а также увидите, как **фильтровать задачи по дате**, чтобы работать только с релевантными элементами вашего расписания.

## Быстрые ответы
- **Что означает «stop» для задачи?** Устанавливает дату остановки, приостанавливая работу после этой даты.  
- **Как возобновить остановленную задачу?** Присвоив дату возобновления, позже даты остановки.  
- **Можно ли ограничить операцию диапазоном дат?** Да — используйте минимальную дату для фильтрации задач.  
- **Какая версия библиотеки требуется?** Любой недавний релиз Aspose.Tasks for Java (API остаётся стабильным).  
- **Нужна ли лицензия для разработки?** Бесплатная пробная версия подходит для тестирования; лицензия требуется для продакшн.

## Что такое «how to resume task» в Aspose.Tasks?
Возобновление задачи просто означает указание **RESUME** даты в объекте `Task`. Когда движок проекта обрабатывает расписание, работа будет продолжаться с этой даты. Это особенно полезно при прерываниях, таких как недоступность ресурсов или внешние зависимости.

## Почему использовать функцию остановки/возобновления?
- **Контроль над графиками:** Приостановить работу без удаления задачи.  
- **Точное отчётность:** Показать реалистичные даты начала/окончания в диаграммах Ганта.  
- **Лёгкая автоматизация:** Сочетать с фильтрами дат для массового обновления задач.

## Предварительные требования
Перед началом убедитесь, что у вас есть:

- Твердое понимание основ Java.  
- Установленный JDK на вашем компьютере.  
- Библиотека Aspose.Tasks for Java, добавленная в ваш проект (скачайте её [здесь](https://releases.aspose.com/tasks/java/)).  

## Импорт пакетов
Сначала импортируйте необходимые классы, чтобы работать с проектами и задачами.

```java
import com.aspose.tasks.ChildTasksCollector;
import com.aspose.tasks.Project;
import com.aspose.tasks.Task;
import com.aspose.tasks.TaskUtils;
import com.aspose.tasks.Tsk;
import java.util.Calendar;
import java.util.GregorianCalendar;
```

## Шаг 1: Инициализация проекта и коллектора
Создайте экземпляр `Project`, указывающий на ваш файл MPP, и настройте `ChildTasksCollector` для сбора всех задач в иерархии.

```java
// The path to the documents directory.
String dataDir = "Your Document Directory";
Project project = new Project(dataDir + "Software Development.mpp");
ChildTasksCollector collector = new ChildTasksCollector();
TaskUtils.apply(project.getRootTask(), collector, 0);
```

## Шаг 2: Определение минимальной даты для фильтрации
Если вы хотите работать только с задачами, у которых есть значимые даты остановки или возобновления, определите **минимальную дату**. Это ядро техники *фильтрации задач по дате*.

```java
java.util.Date minDate = new GregorianCalendar(2000, Calendar.JANUARY, 1).getTime();
```

## Шаг 3: Итерация, проверка и отображение значений Stop/Resume
Теперь пройдитесь по собранным задачам, примените логику **how to stop task** и выведите даты. Код также демонстрирует **how to resume task**, читая свойство `RESUME`.

```java
for (Task tsk : collector.getTasks()) {
    if (tsk.get(Tsk.STOP).before(minDate)) {
        System.out.println("NA");
    } else {
        System.out.println(tsk.get(Tsk.STOP).toString());
    }
    if (tsk.get(Tsk.RESUME).before(minDate)) {
        System.out.println("NA");
    } else {
        System.out.println(tsk.get(Tsk.RESUME).toString());
    }
}
```

> **Pro tip:** Замените вызовы `System.out.println` своей логикой (например, обновлением дат, записью в файл или изменением объектов задач), чтобы действительно *остановить* или *возобновить* задачи по необходимости.

## Распространённые проблемы и решения
| Проблема | Причина | Решение |
|----------|---------|----------|
| `NullPointerException` on `tsk.get(Tsk.STOP)` | У задачи не установлена дата остановки. | Проверьте `null` перед вызовом `.before(minDate)`. |
| Dates appear as `NA` even after setting | Минимальная дата слишком поздняя. | Используйте более раннюю `minDate` или уберите фильтр. |
| Changes not reflected in the saved MPP | Проект не сохранён после изменений. | Вызовите `project.save("output.mpp");` после обновления задач. |

**Additional Q&A**

**В: Как фактически установить новую дату остановки для задачи?**  
**О:** Используйте `tsk.set(Tsk.STOP, new Date(...));` и затем сохраните проект.

**В: Можно ли фильтровать задачи по конкретному диапазону, а не только по минимальной дате?**  
**О:** Да — сравнивайте как `before`, так и `after` с вашими начальной и конечной датами.

**В: Пересчитывает ли API автоматически расписание после изменения дат stop/resume?**  
**О:** После изменения дат вызовите `project.calculateCriticalPath();`, чтобы обновить расписание.
