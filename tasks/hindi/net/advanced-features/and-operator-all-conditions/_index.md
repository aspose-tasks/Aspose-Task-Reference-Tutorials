# Aspose.Tasks के साथ सभी शर्तों में AND Operator का उपयोग

## परिचय

क्या आप अपने प्रोजेक्ट मैनेजमेंट कार्यों को कुशलतापूर्वक सुव्यवस्थित करना चाहते हैं? Aspose.Tasks for .NET के साथ, आप प्रोजेक्ट डेटा को प्रभावी रूप से हेरफेर करने के लिए शक्तिशाली कार्यात्मकताओं का उपयोग कर सकते हैं। इन सुविधाओं में से एक है सभी शर्तों में **use and operator** की क्षमता, जो आपको एक साथ कई मानदंडों के आधार पर कार्यों को फ़िल्टर करने की अनुमति देती है। इस ट्यूटोरियल में, हम आपको इस कार्यक्षमता को चरण-दर-चरण लागू करने की प्रक्रिया के माध्यम से ले जाएंगे, ताकि आप **how to filter tasks** को तेज़ और विश्वसनीय तरीके से कर सकें।

## त्वरित उत्तर
- **AND ऑपरेटर क्या करता है?** यह कई फ़िल्टर शर्तों को संयोजित करता है ताकि केवल *सभी* मानदंडों को पूरा करने वाले कार्य वापस आएँ।  
- **यह सुविधा कौन सी लाइब्रेरी प्रदान करती है?** Aspose.Tasks for .NET.  
- **क्या मुझे लाइसेंस चाहिए?** मूल्यांकन के लिए एक मुफ्त ट्रायल काम करता है; उत्पादन के लिए लाइसेंस आवश्यक है।  
- **क्या मैं MPP फ़ाइल लोड कर सकता हूँ?** हाँ – *.mpp* फ़ाइल लोड करने के लिए `Project` क्लास का उपयोग करें।  
- **क्या सारांश कार्यों को फ़िल्टर करना संभव है?** बिल्कुल, शर्त सूची में `SummaryCondition` जोड़कर।

## “use and operator” सुविधा क्या है?
**use and operator** क्षमता आपको कई `ICondition<T>` इम्प्लीमेंटेशन को `AndAllCondition<T>` के साथ जोड़ने देती है। परिणामी फ़िल्टर केवल उन कार्यों को लौटाता है जो आप द्वारा निर्दिष्ट *हर* शर्त को पूरा करते हैं।

## कई शर्तें क्यों लागू करें?
कई शर्तें लागू करने (उदाहरण के लिए, “कार्य null नहीं है” **and** “कार्य एक सारांश कार्य है”) से आप जिस डेटा के साथ काम करते हैं, उस पर सटीक नियंत्रण मिलता है। यह विशेष रूप से उपयोगी है जब आपको जटिल प्रोजेक्ट शेड्यूल के लिए **create custom filter** लॉजिक बनाने या विशिष्ट कार्य समूहों पर केंद्रित रिपोर्ट उत्पन्न करने की आवश्यकता हो।

## पूर्वापेक्षाएँ

ट्यूटोरियल में डुबकी लगाने से पहले, सुनिश्चित करें कि आपके पास निम्नलिखित पूर्वापेक्षाएँ हैं:

1. **C# का बुनियादी ज्ञान** – C# प्रोग्रामिंग भाषा से परिचित होना लाभदायक होगा।  
2. **Aspose.Tasks for .NET लाइब्रेरी** – Aspose.Tasks for .NET लाइब्रेरी को [here](https://releases.aspose.com/tasks/net/) से डाउनलोड और इंस्टॉल करें।  
3. **इंटीग्रेटेड डेवलपमेंट एनवायरनमेंट (IDE)** – कोडिंग सुविधा के लिए अपने सिस्टम पर Visual Studio जैसे IDE को इंस्टॉल रखें।  

## नेमस्पेस इम्पोर्ट करें

सबसे पहले, आवश्यक क्लास और मेथड तक पहुँचने के लिए आवश्यक नेमस्पेस इम्पोर्ट करने की जरूरत है।

```csharp
using Aspose.Tasks;
using System;
using System.Collections.Generic;

using Aspose.Tasks.Util;
```

अब, प्रक्रिया को स्पष्ट रूप से समझने के लिए उदाहरण को कई चरणों में विभाजित करते हैं।

## चरण 1: प्रोजेक्ट फ़ाइल लोड करें (load mpp file)

```csharp
// The path to the documents directory.
String DataDir = "Your Document Directory";
var project = new Project(DataDir + "Project2.mpp");
```

`Project` क्लास कन्स्ट्रक्टर का उपयोग करके फ़ाइल पाथ निर्दिष्ट कर प्रोजेक्ट फ़ाइल लोड करें। यह चरण **load mpp file** हैंडलिंग को दर्शाता है।

## चरण 2: सभी प्रोजेक्ट कार्य एकत्र करें

```csharp
var coll = new ChildTasksCollector();
TaskUtils.Apply(project.RootTask, coll, 0);
```

प्रोजेक्ट के भीतर सभी कार्यों को एकत्र करने के लिए `ChildTasksCollector` क्लास का उपयोग करें।

## चरण 3: फ़िल्टर शर्तें परिभाषित करें

```csharp
var conditions = new List<ICondition<Task>>
{
    new NotNullCondition(),
    new SummaryCondition()
};
```

कार्य फ़िल्टर करने के लिए शर्तों की एक सूची बनाएं। इस उदाहरण में, हम उन कार्यों को फ़िल्टर करते हैं जो **not null** हैं और **summary tasks** हैं – यह **filter summary tasks** का एक उदाहरण है।

## चरण 4: शर्तों पर AND ऑपरेटर लागू करें (apply multiple conditions)

```csharp
var joinedCondition = new AndAllCondition<Task>(conditions);
```

`AndAllCondition` क्लास का उपयोग करके शर्तों को जोड़ें, सभी शर्तों पर **use and operator** लागू करें।

## चरण 5: कार्य फ़िल्टर करें

```csharp
List<Task> collection = Filter(coll.Tasks, joinedCondition);
```

जुड़ी हुई शर्त को एकत्रित कार्यों पर लागू करके उन्हें उपयुक्त रूप से फ़िल्टर करें। यह चरण संयुक्त शर्त का उपयोग करके **how to filter tasks** दिखाता है।

## चरण 6: फ़िल्टर किए गए कार्यों को प्रोसेस करें

```csharp
foreach (var task in collection)
{
    Console.WriteLine("Name: " + task.Get(Tsk.Name));
    // Perform operations with filtered tasks
}
```

फ़िल्टर किए गए कार्यों पर इटरेट करें और आवश्यकतानुसार ऑपरेशन करें।

## सामान्य समस्याएँ और समाधान

- **Condition सूची खाली है** – `AndAllCondition<T>` बनाने से पहले कम से कम एक `ICondition<T>` जोड़ना सुनिश्चित करें।  
- **कोई कार्य नहीं मिला** – यह जाँचें कि आपने जो शर्तें जोड़ी हैं, वे आपके प्रोजेक्ट के कार्यों से मेल खाती हैं या नहीं (उदाहरण के लिए, आप सारांश कार्यों को फ़िल्टर कर रहे हो सकते हैं जबकि कोई नहीं है)।  
- **फ़ाइल नहीं मिली** – `DataDir` पाथ और *.mpp* फ़ाइल का नाम दोबारा जांचें।  

## अक्सर पूछे जाने वाले प्रश्न

**Q: क्या मैं उदाहरण में दिखाए गए शर्तों के अलावा अतिरिक्त शर्तें लागू कर सकता हूँ?**  
A: हाँ, आप अपने प्रोजेक्ट आवश्यकताओं के आधार पर कोई भी कस्टम शर्तें परिभाषित और लागू कर सकते हैं।

**Q: क्या Aspose.Tasks for .NET विभिन्न प्रोजेक्ट फ़ाइल फ़ॉर्मेट्स के साथ संगत है?**  
A: हाँ, Aspose.Tasks MPP, XML, और CSV जैसे विभिन्न प्रोजेक्ट फ़ाइल फ़ॉर्मेट्स को सपोर्ट करता है।

**Q: क्या Aspose.Tasks जटिल प्रोजेक्ट शेड्यूलिंग एल्गोरिदम के लिए समर्थन प्रदान करता है?**  
A: बिल्कुल, Aspose.Tasks प्रोजेक्ट शेड्यूल को प्रबंधित करने के लिए उन्नत सुविधाएँ प्रदान करता है, जिसमें क्रिटिकल पाथ विश्लेषण और रिसोर्स अलोकेशन शामिल हैं।

**Q: क्या मैं Aspose.Tasks को अन्य .NET फ्रेमवर्क या लाइब्रेरीज़ के साथ एकीकृत कर सकता हूँ?**  
A: हाँ, आप Aspose.Tasks को अन्य .NET फ्रेमवर्क और लाइब्रेरीज़ के साथ सहजता से एकीकृत करके कार्यक्षमता को बढ़ा सकते हैं।

**Q: क्या Aspose.Tasks उपयोगकर्ताओं के लिए कोई कम्युनिटी फ़ोरम या सपोर्ट चैनल उपलब्ध है?**  
A: हाँ, आप किसी भी प्रश्न या सहायता के लिए Aspose.Tasks कम्युनिटी फ़ोरम [here](https://forum.aspose.com/c/tasks/15) तक पहुँच सकते हैं।

## निष्कर्ष

निष्कर्षतः, Aspose.Tasks for .NET के साथ सभी शर्तों में **use and operator** का उपयोग करके आप एक साथ कई मानदंडों के आधार पर प्रोजेक्ट कार्यों को कुशलतापूर्वक फ़िल्टर कर सकते हैं। इस ट्यूटोरियल में प्रदान किए गए चरण‑दर‑चरण मार्गदर्शिका का पालन करके, आप इस कार्यक्षमता को अपने प्रोजेक्ट मैनेजमेंट वर्कफ़्लो में सहजता से एकीकृत कर सकते हैं, जिससे उत्पादकता और संगठन में सुधार होगा।
