# प्रोजेक्ट को PDF के रूप में सहेजें – Aspose.Tasks में टास्क शीट रेंडर करें

## परिचय
इस व्यापक गाइड में आप जानेंगे कि **प्रोजेक्ट को PDF के रूप में कैसे सहेजें** और Aspose.Tasks for Java का उपयोग करके टास्क शीट कैसे जनरेट करें। चाहे आपको स्टेकहोल्डर रिव्यू के लिए MPP फ़ाइल को PDF में एक्सपोर्ट करना हो या रिपोर्टिंग के लिए एक साफ़ टास्क शीट रेंडर करनी हो, यह ट्यूटोरियल हर कदम को समझाता है, प्रत्येक सेटिंग के महत्व को बताता है, और चलाने के लिए आवश्यक सटीक कोड दिखाता है।

## त्वरित उत्तर
- **“save project as pdf” क्या करता है?** यह Microsoft Project फ़ाइल (MPP) को PDF दस्तावेज़ में बदलता है, वैकल्पिक रूप से इसे टास्क शीट के रूप में फ़ॉर्मेट करता है।  
- **टास्क‑शीट PDF के लिए कौन सा फ़ॉर्मेट उपयोग किया जाता है?** `PdfSaveOptions` में `PresentationFormat.TaskSheet` विकल्प।  
- **क्या कोड चलाने के लिए लाइसेंस चाहिए?** उत्पादन के लिए एक टेम्पररी या पूर्ण लाइसेंस आवश्यक है; मूल्यांकन के लिए मुफ्त ट्रायल काम करता है।  
- **मुख्य पूर्वापेक्षाएँ क्या हैं?** Java JDK, Aspose.Tasks for Java लाइब्रेरी, और एक वैध प्रोजेक्ट फ़ाइल (जैसे *.mpp)।  
- **क्या मैं PDF लेआउट को कस्टमाइज़ कर सकता हूँ?** हाँ, Aspose.Tasks कई `PdfSaveOptions` प्रॉपर्टीज़ प्रदान करता है जो स्टाइलिंग और पेज सेटअप को नियंत्रित करती हैं।

## “save project as pdf” क्या है?
प्रोजेक्ट को PDF के रूप में सहेजना आंतरिक प्रोजेक्ट डेटा को एक पोर्टेबल, रीड‑ओनली दस्तावेज़ में बदल देता है। यह उन शेड्यूल, टास्क शीट या स्टेटस रिपोर्ट को शेयर करने के लिए आदर्श है जिनके पास Microsoft Project स्थापित नहीं है।

## टास्क शीट रेंडर क्यों करें?
टास्क शीट प्रत्येक टास्क को टेबलर व्यू में प्रस्तुत करती है, जिसमें आईडी, नाम, शुरू/समाप्ति तिथियाँ और अन्य फ़ील्ड दिखाए जाते हैं। इसे PDF में रेंडर करने से प्लेटफ़ॉर्म के बीच फ़ॉर्मेटिंग सुसंगत रहती है और प्रिंट या अन्य दस्तावेज़ों में एम्बेड करना आसान हो जाता है।

## पूर्वापेक्षाएँ
शुरू करने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित हैं:

1. **Java Development Kit (JDK)** – नवीनतम संस्करण स्थापित और `java`/`javac` आपके PATH में उपलब्ध।  
2. **Aspose.Tasks for Java** – आधिकारिक साइट [यहाँ](https://releases.aspose.com/tasks/java/) से लाइब्रेरी डाउनलोड करें।  
3. एक वैध **प्रोजेक्ट फ़ाइल** (`*.mpp`) जिसे आप कनवर्ट करना चाहते हैं, उदाहरण के लिए `NewProductDev.mpp`।  

## पैकेज इम्पोर्ट करें
सबसे पहले, आवश्यक Aspose.Tasks क्लासेज़ को इम्पोर्ट करें। ये इम्पोर्ट्स आपको PDF सेविंग ऑप्शन्स, प्रेजेंटेशन फ़ॉर्मेट्स, और कोर `Project` ऑब्जेक्ट तक पहुँच देते हैं।

```java
import com.aspose.tasks.PdfSaveOptions;
import com.aspose.tasks.PresentationFormat;
import com.aspose.tasks.Project;
import com.aspose.tasks.SaveOptions;
```

## चरण‑दर‑चरण गाइड

### चरण 1: अपना डॉक्यूमेंट डायरेक्टरी सेट करें
उस फ़ोल्डर को परिभाषित करें जहाँ स्रोत MPP फ़ाइल स्थित है और जहाँ जनरेटेड PDF लिखा जाएगा। एब्सोल्यूट या रिलेटिव पाथ दोनों काम करेंगे; बस यह सुनिश्चित करें कि फ़ोल्डर मौजूद हो।

```java
String dataDir = "Your Document Directory";
```

### चरण 2: प्रोजेक्ट फ़ाइल लोड करें
मौजूदा MPP फ़ाइल को लोड करके एक `Project` इंस्टेंस बनाएं। यह ऑब्जेक्ट मेमोरी में पूरे प्रोजेक्ट स्ट्रक्चर का प्रतिनिधित्व करता है।

```java
Project project = new Project(dataDir + "NewProductDev.mpp");
```

### चरण 3: सेव ऑप्शन कॉन्फ़िगर करें – MPP को PDF में एक्सपोर्ट करें
`PdfSaveOptions` का एक इंस्टेंस बनाएं और Aspose.Tasks को बताएं कि आप PDF को **टास्क शीट** के रूप में रेंडर करना चाहते हैं। यह *export project pdf* उपयोग केस के लिए मुख्य कदम है।

```java
SaveOptions options = new PdfSaveOptions();
options.setPresentationFormat(PresentationFormat.TaskSheet);
```

### चरण 4: प्रोजेक्ट को टास्क शीट के रूप में सहेजें – Save Project as PDF
`Project` ऑब्जेक्ट पर `save` मेथड को कॉल करें, लक्ष्य फ़ाइल नाम और कॉन्फ़िगर किए गए ऑप्शन पास करें। यह कार्रवाई **MPP फ़ाइल को टास्क शीट फ़ॉर्मेट वाले PDF** में बदल देती है।

```java
project.save(dataDir + "taskSheet.pdf", options);
```

### चरण 5: परिणाम की समीक्षा करें
उस डायरेक्टरी (`dataDir`) में जाएँ जिसे आपने निर्दिष्ट किया था। आपको `taskSheet.pdf` दिखना चाहिए जिसमें सभी टास्क की एक साफ़ फ़ॉर्मेटेड सूची होगी। किसी भी PDF व्यूअर से इसे खोलें और पुष्टि करें कि कन्वर्ज़न सफल रहा।

## सामान्य समस्याएँ और समाधान
- **FileNotFoundException** – सुनिश्चित करें कि `dataDir` के अंत में पाथ सेपरेटर (`/` या `\\`) हो और MPP फ़ाइल का नाम सही हो।  
- **LicenseException** – यदि लाइसेंस संबंधी त्रुटि आती है, तो किसी भी Aspose.Tasks API को कॉल करने से पहले एक टेम्पररी या खरीदा हुआ लाइसेंस लागू करें।  
- **Blank PDF** – जाँचें कि प्रोजेक्ट में वास्तव में टास्क मौजूद हैं; खाली प्रोजेक्ट से खाली PDF बनता है।
