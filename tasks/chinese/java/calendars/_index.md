# 如何使用 Aspose.Tasks 创建日历

## 介绍

如果您是一名 Java 开发者，想要 **创建日历** 对象来驱动项目计划，那么您来对地方了。在本中心我们汇集了一系列 Aspose.Tasks 教程，逐步讲解 MS Project 日历管理的各个方面——从创建基本日历到提取工作时间、定义工作日，甚至将日历更新为 MPP 格式。无论您是在构建新的调度引擎还是扩展现有解决方案，这些一步步的指南都能帮助您快速可靠地完成工作。

## 快速答案
- **处理日历的主要类是什么？** `com.aspose.tasks.Calendar`
- **我可以从现有日历中提取工作时间吗？** 是的——使用 `WorkingTime` 集合。
- **是否可以定义自定义工作日？** 当然，可以为每一天设置 `WeekDay` 对象。
- **如何将日历更新为 MPP 格式？** 调用 `project.save("file.mpp", SaveFileFormat.MPP)`。
- **生产环境是否需要许可证？** 非评估部署需要付费的 Aspose.Tasks 许可证。

## 什么是 **创建日历** 在 Aspose.Tasks 中？

在 Aspose.Tasks 中，**创建日历** 是指实例化一个 `Calendar` 对象，将其附加到 `Project`，并配置其工作日、假期和例外。该对象成为 MS Project 文件中所有任务调度计算的核心。

## 为什么使用 Aspose.Tasks 进行 Java 日历管理？

- **完整的 MS Project 兼容性**——日历保存和读取时不会失真。
- **细粒度控制**——可以通过代码调整工作时间、添加例外并操作假期。
- **无需 COM 互操作**——纯 Java，适用于跨平台环境。
- **性能优化的 API**——以最小的内存占用处理大型项目文件。

## 常见问题

**Q: 如何在 Aspose.Tasks 中以编程方式**创建日历**对象？**  
**A:** 实例化 `com.aspose.tasks.Calendar`，通过 `project.getCalendars().add(calendar)` 将其添加到 `Project`，然后配置其 `WeekDays`、`Exceptions` 和 `WorkingTimes`。

**Q: 从现有日历中**提取工作时间**的最佳方法是什么？**  
**A:** 使用 `Calendar.getWorkingTimes()` 集合；每个 `WorkingTime` 条目包含对应日期的开始和结束时间。

**Q: 我可以为自定义工作周**定义工作日**吗？**  
**A:** 可以。通过 `calendar.getWeekDays()` 获取日历中的 `WeekDay` 对象，并为需要自定义的每一天设置 `DayWorking` 和 `WorkingTimes`。

**Q: 更改日历后，有没有简便的方法**更新 mpp**文件？**  
**A:** 修改日历后，调用 `project.save("UpdatedProject.mpp", SaveFileFormat.MPP)` 将更改写回 MPP 文件。

**Q: 每个日历操作都需要单独的许可证吗？**  
**A:** 不需要。单个 Aspose.Tasks 许可证涵盖所有与日历相关的 API，包括创建、提取、定义以及 MPP 更新。
