# مراقبة تكلفة المشروع باستخدام Aspose.Tasks: العمل الإضافي والعمل

## المقدمة
في هذا البرنامج التعليمي ستكتشف كيفية **مراقبة تكلفة المشروع** باستخدام Aspose.Tasks للغة Java. سنستعرض عملية تتبع العمل الإضافي، التكلفة المتبقية، والجهد حتى تتمكن من الحفاظ على مشاريعك ضمن الجدول الزمني والميزانية. سواء كنت مدير مشروع أو قائد فريق، ستساعدك هذه الخطوات على الحفاظ على رؤية واضحة للمعايير المالية والموارد.

## إجابات سريعة
- **ما الذي يمكنني مراقبته؟** تكلفة العمل الإضافي، عمل العمل الإضافي، التكلفة المتبقية، العمل المتبقي، وتكلفة العمل الإضافي المتبقية.  
- **ما المكتبة المطلوبة؟** Aspose.Tasks للغة Java.  
- **هل أحتاج إلى ترخيص للتطوير؟** النسخة التجريبية المجانية تعمل للاختبار؛ الترخيص مطلوب للإنتاج.  
- **هل يمكنني تحميل ملفات .mpp موجودة؟** نعم، ما عليك سوى توفير مسار الملف.  
- **هل Java 6 كافية؟** تدعم الواجهة البرمجية Java SE 6 وما بعده.  

## ما هي مراقبة تكلفة المشروع؟
مراقبة تكلفة المشروع هي ممارسة تتبع جميع الجوانب المالية للمشروع بشكل مستمر — التكاليف المت预算ة، المصروفات الفعلية، والتكاليف المتبقية المتوقعة. من خلال دمج ذلك مع تعيينات الموارد، تحصل على رؤية فورية للنفقات المتعلقة بالعمل الإضافي وتقدم العمل.

## لماذا نراقب العمل الإضافي والعمل المتبقي؟
- **التحكم في الميزانيات:** غالبًا ما يؤدي العمل الإضافي إلى تجاوزات تكلفة غير متوقعة.  
- **تحسين التنبؤ:** معرفة العمل المتبقي تساعدك على تعديل الجداول بشكل استباقي.  
- **زيادة الشفافية:** يمكن لأصحاب المصلحة رؤية بالضبط أين تُستهلك الموارد.  

## المتطلبات المسبقة
قبل أن نبدأ، تأكد من توفر ما يلي:
1. **مجموعة تطوير جافا (JDK):** تتطلب Aspose.Tasks للغة Java Java SE 6 أو أحدث.  
2. **مكتبة Aspose.Tasks للغة Java:** قم بتحميل وتثبيت المكتبة من [here](https://releases.aspose.com/tasks/java/).  
3. **بيئة تطوير متكاملة (IDE):** أي IDE لجافا مثل Eclipse أو IntelliJ IDEA أو NetBeans.  

## استيراد الحزم
ابدأ باستيراد الحزم اللازمة في ملف Java الخاص بك:
```java
import com.aspose.tasks.Asn;
import com.aspose.tasks.Project;
import com.aspose.tasks.ResourceAssignment;
```

## الخطوة 1: إعداد دليل البيانات
حدد الدليل الذي يقع فيه ملف المشروع الخاص بك:
```java
String dataDir = "Your Data Directory";
```
استبدل `"Your Data Directory"` بالمسار إلى ملف المشروع الخاص بك.

## الخطوة 2: تحميل المشروع
أنشئ كائن `Project` وحمّل ملف المشروع:
```java
Project project = new Project(dataDir + "ResourceAssignmentOvertimes.mpp");
```
استبدل `"ResourceAssignmentOvertimes.mpp"` باسم ملف MPP الخاص بك. توضح هذه الخطوة استخدام **load mpp file**.

## الخطوة 3: التكرار عبر تعيينات الموارد
قم بالتكرار عبر كل تعيين مورد في المشروع:
```java
for (ResourceAssignment ra : project.getResourceAssignments()) {
```

## الخطوة 4: طباعة تكاليف العمل الإضافي والجهد
استخرج واطبع تكاليف العمل الإضافي والجهد لكل تعيين مورد:
```java
    System.out.println(ra.get(Asn.OVERTIME_COST));
    System.out.println(ra.get(Asn.OVERTIME_WORK).toString());
```

## الخطوة 5: طباعة التكاليف المتبقية والجهد
استخرج واطبع التكاليف المتبقية والجهد لكل تعيين مورد:
```java
    System.out.println(ra.get(Asn.REMAINING_COST));
    System.out.println(ra.get(Asn.REMAINING_WORK).toString());
```

## الخطوة 6: طباعة تكاليف العمل الإضافي المتبقية والجهد
استخرج واطبع تكاليف العمل الإضافي المتبقية والجهد لكل تعيين مورد:
```java
    System.out.println(ra.get(Asn.REMAINING_OVERTIME_COST));
    System.out.println(ra.get(Asn.REMAINING_OVERTIME_WORK).toString());
}
```

## المشكلات الشائعة والحلول
- **الملف غير موجود:** تحقق مرة أخرى من مسار `dataDir` وتأكد من صحة اسم ملف MPP.  
- **قيم فارغة:** قد لا تحتوي بعض التعيينات على بيانات عمل إضافي؛ احرص على التعامل مع `null` عند الطباعة.  
- **عدم توافق الإصدارات:** استخدم نسخة المكتبة التي تتوافق مع تنسيق ملف MPP (مثل إصدارات MS Project الأحدث).
