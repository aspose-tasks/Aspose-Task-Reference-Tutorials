# การตรวจสอบต้นทุนโครงการด้วย Aspose.Tasks: เวลาทำงานล่วงเวลาและงาน

## บทนำ
ในบทแนะนำนี้คุณจะได้เรียนรู้วิธี **การตรวจสอบต้นทุนโครงการ** ด้วย Aspose.Tasks for Java เราจะอธิบายขั้นตอนการติดตามเวลาทำงานล่วงเวลา, ต้นทุนที่เหลือ, และงาน เพื่อให้คุณสามารถรักษาโครงการให้เป็นไปตามกำหนดและอยู่ในงบประมาณ ไม่ว่าคุณจะเป็นผู้จัดการโครงการหรือหัวหน้าทีม ขั้นตอนเหล่านี้จะช่วยให้คุณมองเห็นสถานะทางการเงินและทรัพยากรได้อย่างชัดเจน

## คำตอบสั้น
- **ฉันสามารถตรวจสอบอะไรได้บ้าง?** ต้นทุนเวลาทำงานล่วงเวลา, งานล่วงเวลา, ต้นทุนที่เหลือ, งานที่เหลือ, และต้นทุนล่วงเวลาที่เหลือ.  
- **ต้องใช้ไลบรารีอะไร?** Aspose.Tasks for Java.  
- **ต้องใช้ไลเซนส์สำหรับการพัฒนาหรือไม่?** สามารถใช้รุ่นทดลองฟรีสำหรับการทดสอบ; จำเป็นต้องมีไลเซนส์สำหรับการใช้งานจริง.  
- **สามารถโหลดไฟล์ .mpp ที่มีอยู่ได้หรือไม่?** ได้, เพียงระบุเส้นทางไปยังไฟล์.  
- **Java 6 เพียงพอหรือไม่?** API รองรับ Java SE 6 ขึ้นไป.

## การตรวจสอบต้นทุนโครงการคืออะไร?
การตรวจสอบต้นทุนโครงการคือการติดตามอย่างต่อเนื่องทุกด้านทางการเงินของโครงการ—ต้นทุนที่กำหนดไว้ในงบประมาณ, รายจ่ายจริง, และต้นทุนที่คาดว่าจะเหลืออยู่ การรวมข้อมูลนี้กับการมอบหมายทรัพยากรทำให้คุณได้รับข้อมูลเชิงลึกแบบเรียลไทม์เกี่ยวกับค่าใช้จ่ายล่วงเวลาและความคืบหน้าของงาน.

## ทำไมต้องตรวจสอบเวลาทำงานล่วงเวลาและงานที่เหลือ?
- **ควบคุมงบประมาณ:** เวลาทำงานล่วงเวลามักทำให้เกิดค่าใช้จ่ายเกินงบประมาณที่ไม่คาดคิด.  
- **ปรับปรุงการคาดการณ์:** การรู้ว่ามีงานเหลือเท่าไหร่ช่วยให้คุณปรับแผนกำหนดเวลาได้อย่างเชิงรุก.  
- **เพิ่มความโปร่งใส:** ผู้มีส่วนได้ส่วนเสียสามารถเห็นได้อย่างชัดเจนว่าทรัพยากรถูกใช้ที่ไหน.

## ข้อกำหนดเบื้องต้น
ก่อนที่เราจะเริ่ม, โปรดตรวจสอบว่าคุณมีสิ่งต่อไปนี้:
1. **Java Development Kit (JDK):** Aspose.Tasks for Java ต้องการ Java SE 6 หรือใหม่กว่า.  
2. **Aspose.Tasks for Java Library:** ดาวน์โหลดและติดตั้งไลบรารีจาก [ที่นี่](https://releases.aspose.com/tasks/java/).  
3. **Integrated Development Environment (IDE):** IDE ของ Java ใดก็ได้ เช่น Eclipse, IntelliJ IDEA หรือ NetBeans.

## นำเข้าแพ็กเกจ
เริ่มต้นโดยการนำเข้าแพ็กเกจที่จำเป็นในไฟล์ Java ของคุณ:
```java
import com.aspose.tasks.Asn;
import com.aspose.tasks.Project;
import com.aspose.tasks.ResourceAssignment;
```

## ขั้นตอนที่ 1: ตั้งค่าไดเรกทอรีข้อมูล
กำหนดไดเรกทอรีที่ไฟล์โครงการของคุณตั้งอยู่:
```java
String dataDir = "Your Data Directory";
```
แทนที่ `"Your Data Directory"` ด้วยเส้นทางไปยังไฟล์โครงการของคุณ.

## ขั้นตอนที่ 2: โหลดโครงการ
สร้างอ็อบเจกต์ `Project` และโหลดไฟล์โครงการ:
```java
Project project = new Project(dataDir + "ResourceAssignmentOvertimes.mpp");
```
แทนที่ `"ResourceAssignmentOvertimes.mpp"` ด้วยชื่อไฟล์ MPP ของคุณ ขั้นตอนนี้แสดงการใช้ **load mpp file**.

## ขั้นตอนที่ 3: วนลูปผ่านการมอบหมายทรัพยากร
วนลูปผ่านการมอบหมายทรัพยากรแต่ละรายการในโครงการ:
```java
for (ResourceAssignment ra : project.getResourceAssignments()) {
```

## ขั้นตอนที่ 4: พิมพ์ต้นทุนล่วงเวลาและงาน
ดึงและพิมพ์ต้นทุนล่วงเวลาและงานสำหรับการมอบหมายทรัพยากรแต่ละรายการ:
```java
    System.out.println(ra.get(Asn.OVERTIME_COST));
    System.out.println(ra.get(Asn.OVERTIME_WORK).toString());
```

## ขั้นตอนที่ 5: พิมพ์ต้นทุนที่เหลือและงานที่เหลือ
ดึงและพิมพ์ต้นทุนที่เหลือและงานที่เหลือสำหรับการมอบหมายทรัพยากรแต่ละรายการ:
```java
    System.out.println(ra.get(Asn.REMAINING_COST));
    System.out.println(ra.get(Asn.REMAINING_WORK).toString());
```

## ขั้นตอนที่ 6: พิมพ์ต้นทุนล่วงเวลาที่เหลือและงานที่เหลือ
ดึงและพิมพ์ต้นทุนล่วงเวลาที่เหลือและงานที่เหลือสำหรับการมอบหมายทรัพยากรแต่ละรายการ:
```java
    System.out.println(ra.get(Asn.REMAINING_OVERTIME_COST));
    System.out.println(ra.get(Asn.REMAINING_OVERTIME_WORK).toString());
}
```

## ปัญหาทั่วไปและวิธีแก้
- **ไฟล์ไม่พบ:** ตรวจสอบเส้นทาง `dataDir` อีกครั้งและให้แน่ใจว่าชื่อไฟล์ MPP ถูกต้อง.  
- **ค่า null:** การมอบหมายบางรายการอาจไม่มีข้อมูลล่วงเวลา; ตรวจสอบ `null` ก่อนพิมพ์.  
- **เวอร์ชันไม่ตรงกัน:** ใช้เวอร์ชันของไลบรารีที่ตรงกับรูปแบบไฟล์ MPP (เช่น เวอร์ชันใหม่ของ MS Project).
