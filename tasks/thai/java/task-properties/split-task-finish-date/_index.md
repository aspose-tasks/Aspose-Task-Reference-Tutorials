# แยกวันที่สิ้นสุดของงานใน Aspose.Tasks

## บทนำ
การ **จัดการไทม์ไลน์ของโครงการ** อย่างมีประสิทธิภาพเป็นหัวใจสำคัญของการส่งมอบโครงการที่ประสบความสำเร็จ เมื่อระยะเวลาของงานเปลี่ยนแปลง วันที่สิ้นสุดของงานต้องถูกคำนวณใหม่เพื่อให้ตารางเวลาแม่นยำ ในบทแนะนำนี้เราจะสาธิต **วิธีการแยกวันที่สิ้นสุดของงาน** ด้วย Aspose.Tasks สำหรับ Java เพื่อให้คุณควบคุมไทม์ไลน์ของโครงการได้อย่างแม่นยำ

## คำตอบสั้น
- **การแยกวันที่สิ้นสุดของงานทำให้ได้อะไร?** ช่วยให้คุณคำนวณวันที่สิ้นสุดสำหรับระยะเวลาใด ๆ ได้ ทำให้สามารถปรับตารางเวลาได้ทันที  
- **ต้องใช้ไลบรารีอะไร?** Aspose.Tasks for Java (ดาวน์โหลดได้จากเว็บไซต์ทางการ)  
- **ต้องใช้ไลเซนส์สำหรับการพัฒนาหรือไม่?** ไลเซนส์ชั่วคราวใช้สำหรับการทดสอบได้; ต้องมีไลเซนส์เต็มสำหรับการใช้งานจริง  
- **สามารถใช้กับปฏิทินโครงการใดก็ได้หรือไม่?** ใช่, API จะใช้ปฏิทินของโครงการเพื่อคำนึงถึงวันทำงานและวันหยุด  
- **ต้องใช้โค้ดกี่บรรทัด?** น้อยกว่า 10 บรรทัดเพื่อรับวันที่สิ้นสุดสำหรับระยะเวลาใด ๆ

## การ “จัดการไทม์ไลน์ของโครงการ” หมายถึงอะไรในบริบทของ Aspose.Tasks?
การจัดการไทม์ไลน์ของโครงการหมายถึงการทำให้วันที่เริ่มต้นและสิ้นสุดของทุกงานสอดคล้องกับตารางโดยรวม โดยคำนึงถึงปฏิทิน, ทรัพยากร, และความขึ้นต่อกันของงาน การคำนวณวันที่สิ้นสุดที่แม่นยำเป็นสิ่งจำเป็นสำหรับการคาดการณ์ที่เป็นจริงและความเชื่อมั่นของผู้มีส่วนได้ส่วนเสีย

## ทำไมต้องแยกวันที่สิ้นสุดของงาน?
- **ความยืดหยุ่น:** ดูผลของการจัดสรรชั่วโมงทำงานที่ต่างกันต่อการส่งมอบได้อย่างรวดเร็ว  
- **การประเมินความเสี่ยง:** ประเมินสถานการณ์ “ถ้าเป็นอย่างนี้” โดยไม่ต้องแก้ไขแผนเดิม  
- **การวางแผนทรัพยากร:** ปรับระยะเวลาของงานให้สอดคล้องกับความสามารถของทีมและข้อจำกัดของปฏิทิน

## ข้อกำหนดเบื้องต้น
ก่อนเริ่ม, โปรดตรวจสอบว่าคุณมีสิ่งต่อไปนี้:
- ความรู้พื้นฐานของการเขียนโปรแกรม Java  
- ไลบรารี Aspose.Tasks for Java ติดตั้งแล้ว คุณสามารถดาวน์โหลดได้ [ที่นี่](https://releases.aspose.com/tasks/java/)  
- ตั้งค่าสภาพแวดล้อมการพัฒนา Java เรียบร้อย

## นำเข้าแพ็กเกจ
เริ่มต้นโดยรวมแพ็กเกจที่จำเป็นในโครงการ Java ของคุณ:
```java
import com.aspose.tasks.*;
```

## ขั้นตอนที่ 1: ค้นหางานที่ต้องการแยก
ค้นหางานที่คุณต้องการแยกภายในโครงการ:
```java
// The path to the documents directory.
String dataDir = "Your Document Directory";
// Read project
String projectName = dataDir + "SplitTask.mpp";
Project project = new Project(projectName);
Task splitTask = project.getRootTask().getChildren().getByUid(1);
```

## ขั้นตอนที่ 2: ค้นหาปฏิทินของโครงการ
ดึงปฏิทินของโครงการเพื่อคำนวณวันที่อย่างแม่นยำ:
```java
Calendar calendar = project.get(Prj.CALENDAR);
```

## ขั้นตอนที่ 3: คำนวณวันที่สิ้นสุดของงานด้วยระยะเวลาต่าง ๆ
ตอนนี้ให้คำนวณวันที่สิ้นสุดของงานสำหรับระยะเวลาต่าง ๆ

### ขั้นตอนย่อย 3.1: ระยะเวลา 8 ชั่วโมง
```java
System.out.println("Start Date: " + splitTask.get(Tsk.START) + "\n+ Duration 8 hours\nFinish Date: " + calendar.getTaskFinishDateFromDuration(splitTask, 8d));
```

*ทำซ้ำโค้ดด้านบนสำหรับระยะเวลาที่ต่างกันโดยปรับจำนวนชั่วโมงตามต้องการ เพื่อดูว่าการเปลี่ยนแปลงแต่ละครั้งส่งผลต่อวันที่สิ้นสุดอย่างไร*

## ปัญหาที่พบบ่อยและวิธีแก้ไข
- **การอ้างอิงปฏิทินไม่ถูกต้อง:** ตรวจสอบให้แน่ใจว่าคุณดึงปฏิทินของโครงการ (`project.get(Prj.CALENDAR)`) ไม่ได้สร้างใหม่  
- **UID ของงานไม่ถูกต้อง:** UID ต้องตรงกับงานที่มีอยู่จริง; หากไม่ตรง `getByUid` จะคืนค่า `null` ทำให้เกิด `NullPointerException`  
- **ความไม่ตรงกันของเขตเวลา:** API ทำงานตามเขตเวลาของปฏิทิน; ตรวจสอบเขตเวลาตั้งต้นของระบบหากผลลัพธ์ดูแปลก

## คำถามที่พบบ่อยเพิ่มเติม

**Q: ฉันสามารถใช้เทคนิคนี้กับปฏิทินที่กำหนดเองได้หรือไม่?**  
A: แน่นอน เพียงแทนที่ `project.get(Prj.CALENDAR)` ด้วยอินสแตนซ์ `Calendar` ที่คุณกำหนดเอง

**Q: วิธีการนี้คำนึงถึงวันไม่ทำงานหรือไม่?**  
A: ใช่, นิยามเวลาทำงานของปฏิทินจะถูกนำมาใช้เมื่อคำนวณวันที่สิ้นสุด

**Q: มีวิธีดึงวันที่สิ้นสุดสำหรับชั่วโมงเศษส่วน (เช่น 3.5 ชั่วโมง) หรือไม่?**  
A: ส่งค่า `double` เช่น `3.5d` ไปยัง `getTaskFinishDateFromDuration`; API จะจัดการกับระยะเวลาเศษส่วนได้

**Q: ฉันจะจัดรูปแบบวันที่ผลลัพธ์อย่างไร?**  
A: ใช้ `java.time.format.DateTimeFormatter` กับอ็อบเจกต์ `Date` ที่คืนค่าเพื่อแสดงในรูปแบบที่คุณต้องการ
